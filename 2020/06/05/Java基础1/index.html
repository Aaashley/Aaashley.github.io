1.JDK,JRE,JVM

  • JVM（Java Virtual Machine） java虚拟机，运行程序时，jvm负责将字节码转换为特定机器代码，提供了内存管理/垃圾回收和安全机制。
  • JRE（Java Runtime Environment） java运行时环境，jvm的实现，提供了运行java程序的平台（jvm+基础类库）
  • JDK（Java Development Kit）开发工具包，是Java开发环境的核心组件，并且提供编译、调试和运行一个Java程序所需要的所有工具，可执行文件和二进制文件，是一个平台特定的软件。（jvm+基础类库+编译工具）

Java的跨平台性：一次编写多处执行

  1. jvm屏蔽了操作系统底层硬件差异
  2. java面向jvm编程，先编译生成字节码文件（javac生成 .class文件 ），然后交给jvm解释成机器码（java 运行.class文件）执行。

2.抽象类和接口的区别

抽象类 接口
关键字 extends Implements
单继承 实现多个
抽象方法 可以没有，可以抽象+非抽象 都是抽象方法
成员变量 可以存在普通成员变量 Static final且必须初始化，只有常量没有变量
构造方法 可以有 无
抽象方法 public,protected,default 默认必须public

当我们仅仅需要定义一些抽象方法而不需要其余额外的具体方法或者变量的时候，我们可以使用接口。反之，则需要使用抽象类，因为抽象类中可以有非抽象方法和变量。

3. 8种基本数据类型及取值

  1. byte: 1字节-128~127
  2. Short: 2字节 -2^15~2^15-1
  3. Int:4字节-2^31~2^31-1
  4. long:8字节 -2^63~2^63-1
  5. boolean:1字节 true false
  6. float:4字节 -3.403E38~3.403E38
  7. double:8字节 -1.798E308~- 4.9E324
  8. Char:2字节 ’\u0000‘‘ ’\uffff ‘(16进制的，换算过来即065535)
  • bit(位)：二进制位，计算机内部数据储存的最小单位
  • 字节（byte):一个字节由八个二进制位构成，即1个字节等于8个比特（1Byte=8bit）

4.反射机制

在运行中，对于任意一个类，都能知道这个类的所有属性和方法，对于任意一个对象，都能调用它的任意一个方法和属性，即动态获取信息和动态调用对象方法的功能称为反射机制。

  • 获取Class类的三种基本方式：

    1.通过类名称.class来获取(类名的属性)

    2.通过对象.getclass()方法来获取Class类对象

    3.通过类名称加载类Class.forName(“全类名”)，将字节码文件加载进内存，返回class对象。

同一个字节码文件（*.class)在一次程序运行过程中只会被加载一次，无论通过哪种方式获取的class对象都是一个
